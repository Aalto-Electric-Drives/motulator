class motulator.drive.model.InductionMachine(par)[source]#

Bases: motulator.common.model.Subsystem

-equivalent model of an induction machine.

An induction machine is modeled using the -equivalent model [1]. The stator inductance L_s can either be constant or a function of the stator flux magnitude:

L_s = L_s(abs(psi_s_ab))
Parameters:

par (InductionMachinePars | InductionMachineInvGammaPars) Machine parameters.

Notes

The model is chosen here since it can be extended with the magnetic saturation model in a straightforward manner. If the magnetic saturation is omitted, the model is mathematically identical to the inverse- and T models [1].

References

[1] (1,2)

Slemon, Modelling of induction machines for electric drives, IEEE Trans. Ind. Appl., 1989, https://doi.org/10.1109/28.44251

class motulator.drive.model.InductionMachinePars[source]#

-model parameters of an induction machine.

This contains -model parameters of an induction machine. The main-flux saturation saturation can also be modeled by providing a callable L_s parameter. For convenience, the class also provides the corresponding inverse- model parameters, which can be used in control systems. If the saturation is modeled, these inverse- parameters depend on the stator flux linkage magnitude psi_s that should be updated by calling the update_psi_s method.

  • R_s (float) Stator resistance ().

  • R_r (float) Rotor resistance ().

  • L_ell (float) Leakage inductance (H).

  • L_s (float | Callable[[float], float]) Stator inductance (H). If callable, it should be a function of the stator flux linkage magnitude (Vs).

class motulator.drive.model.SaturatedSynchronousMachinePars[source]#

Bases: BaseSynchronousMachinePars

Parameters of a saturated synchronous machine.

The saturation model is specified as a current map (current as a function of the flux linkage). Optionally, to be used only in control systems, a flux map (flux linkage as a function of the current) can be provided. For convenience, this class also provides the incremental inductance matrix and its inverse, which can be used in control systems and optimal reference generation.

Parameters:

  • n_p (int) Number of pole pairs.

  • R_s (float) Stator resistance ().

  • i_s_dq_fcn (Callable[[complex], complex], optional) Stator current (A) as a function of the stator flux linkage (Vs). This function should be differentiable, if inverse incremental inductances are used. Needed in the system model and in some control methods.

  • psi_s_dq_fcn (Callable[[complex], complex], optional) Stator flux linkage (Vs) as a function of the stator current (A). This function should be differentiable, if incremental inductances are used. Needed only for some control methods, not in the system model.

  • kind (str, optional) Machine type, defaults to pm. Allowed values are pm (permanent magnet) and rel (reluctance).

  • max_iter (int, optional) Maximum number of iterations, defaults to None. Value around 20 typically suffices. Note that the iterative method is intended for development purposes.

Notes

The class allows providing either i_s_dq_fcn or psi_s_dq_fcn. If only one of them is provided and max_iter is given, the other one is computed iteratively. This feature is intended for development purposes. It can be used in control systems, but iteration increases the simulations time and may not be computationally practical in real-time control.
